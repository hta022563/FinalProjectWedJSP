##  QUY ĐỊNH LÀM VIỆC (TEAM RULES) - QUAN TRỌNG!

Để tránh xung đột code (Conflict) và quản lý dự án hiệu quả, tất cả thành viên bắt buộc tuân thủ:

### 1. Quy tắc đặt tên nhánh (Branching)
- Tuyệt đối **KHÔNG** push trực tiếp lên nhánh `main`.
- Đặt tên nhánh theo cú pháp: `feature/[tên-chức-năng]`
  - Ví dụ: `feature/admin-dashboard`, `feature/payment-method`.

### 2. Quy trình gộp code (Git Workflow)
1. Trước khi code: `git pull origin main` để cập nhật code mới nhất.
2. Code xong: Push lên nhánh cá nhân và tạo **Pull Request (PR)** trên GitHub.
3. **Hảo OR 1 thành viên khác sẽ Approve và Merge PR vào nhánh main.**
4. Sau khi PR được gộp, hãy xóa nhánh cá nhân để tránh rác Repo.(chưa cần thiết, cứ giữ nguyên)
